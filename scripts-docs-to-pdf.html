Documentation to PDF Scripts

This directory contains scripts to combine all markdown documentation files into a single PDF, organized by category.

Categories

Documents are automatically categorized into four groups:

  1. End-User Big Picture: READMEs, quickstarts, getting started guides
  2. End-User Details: User guides, tutorials, how-tos, showcase/demo docs
  3. Other Big Picture: Architecture, deployment overviews, system design
  4. Other Details: Implementation details, compliance docs, operations

Quick Start

Option 1: Simple Markdown Combiner (‚úÖ RECOMMENDED)

Combines all docs into a single markdown file, which you can then convert to PDF:

# Generate combined markdown
npx tsx scripts/combine-docs-simple.ts

# Convert to PDF using pandoc (if installed)
pandoc care-commons-documentation.md -o care-commons-documentation.pdf --toc --pdf-engine=xelatex -V geometry:margin=1in

# Or specify custom output name
npx tsx scripts/combine-docs-simple.ts my-docs.md

Why recommended: More reliable, easier to troubleshoot, and you can preview the markdown before converting.

Option 2: Direct PDF Generation (Advanced)

‚ö†Ô∏è Note: This option requires additional LaTeX packages for SVG support and may fail if documentation contains SVG diagrams. Use Option 1 for most reliable results.

# Generate PDF directly
npx tsx scripts/combine-docs-to-pdf.ts

# Or specify custom output name
npx tsx scripts/combine-docs-to-pdf.ts my-docs.pdf

If you encounter SVG-related errors, install librsvg:

# macOS
brew install librsvg

# Ubuntu/Debian
sudo apt-get install librsvg2-bin

# Then retry the conversion

Installation Requirements

For Markdown Combiner (Option 1)

No dependencies required! The script just combines markdown files.

To convert the markdown to PDF, you have several options:

Pandoc (Best quality):

# macOS
brew install pandoc basictex

# Ubuntu/Debian
sudo apt-get install pandoc texlive-xetex

# Windows
choco install pandoc miktex

Markdown-PDF (Simpler):

npm install -g markdown-pdf
markdown-pdf combined.md

Online Converter (No installation):

  • Upload to https://cloudconvert.com/md-to-pdf
  • Or https://www.markdowntopdf.com/

For Direct PDF Generation (Option 2)

Requires pandoc and LaTeX:

# macOS
brew install pandoc basictex
# Then add LaTeX to PATH:
eval "$(/usr/libexec/path_helper)"

# Ubuntu/Debian
sudo apt-get install pandoc texlive-xetex texlive-fonts-recommended

# Windows
choco install pandoc miktex

Output

The combined documentation includes:

  • Title page with generation timestamp
  • Table of contents (when using pandoc)
  • Part 1: End-User Overview - High-level guides and quickstarts
  • Part 2: End-User Guides - Detailed user documentation
  • Part 3: System Architecture & Deployment - Technical overviews
  • Part 4: Implementation Details - Code, compliance, operations

Each document is separated by page breaks and includes its relative path as a heading.

Customization

To modify categorization logic, edit the categorizeFile() function in either script:

function categorizeFile(filePath: string, content: string): { 
  category: 'end-user-big-picture' | 'end-user-details' | 'other-big-picture' | 'other-details',
  priority: number 
}

Examples

# Basic usage - creates care-commons-documentation.md
npx tsx scripts/combine-docs-simple.ts

# Custom filename
npx tsx scripts/combine-docs-simple.ts project-docs.md

# Convert to PDF with custom settings
pandoc combined.md -o output.pdf \
  --toc \
  --toc-depth=3 \
  --pdf-engine=xelatex \
  -V geometry:margin=0.75in \
  -V fontsize=10pt \
  -V documentclass=report

Troubleshooting

"pandoc: command not found"

Install pandoc using the commands above, or use the markdown combiner and convert online.

"xelatex not found"

Install a LaTeX distribution (BasicTeX, TeX Live, or MiKTeX) as shown above.

SVG image conversion errors

If you see errors like "Could not convert image ... rsvg-convert ... does not exist":

# macOS
brew install librsvg

# Ubuntu/Debian
sudo apt-get install librsvg2-bin

Alternatively, use the simple combiner (Option 1) and convert online at CloudConvert which handles SVGs natively.

LaTeX package missing (svg.sty)

If you see "File `svg.sty' not found", install the full LaTeX distribution:

# macOS - upgrade from BasicTeX to full distribution
brew install --cask mactex

# Ubuntu/Debian
sudo apt-get install texlive-full

Or use Option 1 and convert via an online service.

PDF is too large

The combined markdown can be quite large (~2MB, generates ~500+ page PDF). Consider:

  • Filtering specific directories
  • Excluding implementation details
  • Splitting into multiple PDFs by category

Fonts or encoding issues

Use xelatex engine (default in scripts) which has better Unicode support:

pandoc input.md -o output.pdf --pdf-engine=xelatex

Script Details

combine-docs-simple.ts

  • Purpose: Combines markdown files into a single markdown document
  • Output: .md file
  • Dependencies: None (uses Node.js built-ins)
  • Recommended: Best for flexibility and troubleshooting

combine-docs-to-pdf.ts

  • Purpose: Combines markdown and generates PDF in one step
  • Output: .pdf file
  • Dependencies: Requires pandoc and LaTeX
  • Recommended: Best when you have dependencies installed

File Exclusions

Both scripts automatically exclude:

  • node_modules/
  • .git/
  • dist/, build/, .next/
  • coverage/
  • .turbo/, .vercel/

To modify exclusions, edit the findMarkdownFiles() function.
