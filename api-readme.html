API Directory - Vercel Serverless Functions

This directory contains the entry point for Vercel serverless function deployment.

Security Configuration

‚úÖ Helmet CSP

Configured in: packages/app/src/server.ts

app.use(helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      styleSrc: ["'self'", "'unsafe-inline'"],
      scriptSrc: ["'self'"],
      imgSrc: ["'self'", "data:", "https:"],
    },
  },
}));
  • ‚úÖ CSP is enabled (not disabled)
  • ‚úÖ Proper directives configured
  • ‚úÖ Follows security best practices

‚úÖ CORS Configuration

Configured in: packages/app/src/server.ts

// Development: Allow all origins
// Production: Require CORS_ORIGIN environment variable

const allowedOrigins = process.env['CORS_ORIGIN']?.split(',') ?? [];

app.use(cors({
  origin: (origin, callback) => {
    if (typeof origin !== 'string') {
      callback(null, true); // Allow non-browser requests
      return;
    }

    if (NODE_ENV === 'development') {
      callback(null, true); // Allow all in development
      return;
    }

    // Production: Only allow configured origins
    if (allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'), false);
    }
  },
  credentials: true,
}));
  • ‚úÖ No wildcard (*) in production
  • ‚úÖ Requires explicit origin configuration
  • ‚úÖ Development-friendly (allows all in dev mode)

How It Works

  1. Request Flow:

    Vercel ‚Üí api/index.ts ‚Üí createApp() ‚Üí Express App (with security middleware)

  2. Security Layers:

    • Helmet CSP (Content Security Policy)
    • CORS (Cross-Origin Resource Sharing)
    • Request validation
    • Error handling

  3. Caching:

    • Express app is initialized once per cold start
    • Subsequent requests reuse the same app instance (warm starts)
    • Efficient for serverless environments

Files

  • index.ts - Serverless function handler (TypeScript)
  • README.md - This file

Important Notes

‚ö†Ô∏è Do NOT create index.js in this directory

  • The TypeScript file (index.ts) is the source of truth
  • JavaScript files should not exist here
  • Vercel compiles TypeScript during deployment

‚ö†Ô∏è Security configuration is in packages/app/src/server.ts

  • Do not configure Helmet or CORS in this file
  • This handler simply forwards requests to the Express app
  • Security middleware is applied by the Express app

Testing Locally

# Build the project
npm run build

# Start the dev server (not Vercel)
npm run dev

# Test API endpoints
curl http://localhost:3000/health
curl http://localhost:3000/api

Deployment

Vercel automatically:

  1. Detects api/index.ts
  2. Compiles TypeScript to JavaScript
  3. Creates a serverless function
  4. Routes requests to the function

No additional configuration needed!
