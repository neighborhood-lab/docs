Documentation Site Generator

This script generates a beautiful, searchable static documentation website from all markdown files in the repository.

Overview

The documentation site generator creates a GitHub Pages-ready static website with:

  • üì± Responsive design - Works on desktop, tablet, and mobile
  • üîç Built-in search - Search across all documentation
  • üß≠ Smart navigation - Organized by category with collapsible sections
  • üé® Modern UI - Clean, professional design with syntax highlighting
  • ‚ö° Fast & lightweight - Pure HTML/CSS/vanilla JS, no frameworks
  • üìö Auto-categorization - Intelligently organizes docs into 4 categories

Quick Start

Generate the Site

# Generate documentation site
npx tsx scripts/generate-docs-site.ts

# Output will be in: docs-site/

Preview Locally

# Navigate to output directory
cd docs-site/

# Start a local web server
python3 -m http.server 8000

# Open in browser
open http://localhost:8000

Deploy to GitHub Pages

The docs-site/ directory is git-ignored in this repository. To deploy:

Option 1: Separate Docs Repository (Recommended)

# 1. Create a new GitHub repository (e.g., "care-commons-docs")
gh repo create care-commons-docs --public

# 2. Copy the generated site
cp -r docs-site/* /path/to/care-commons-docs/

# 3. Commit and push
cd /path/to/care-commons-docs/
git add .
git commit -m "Update documentation site"
git push origin main

# 4. Enable GitHub Pages
# Go to repository Settings ‚Üí Pages
# Select "Deploy from a branch"
# Choose "main" branch and "/ (root)" folder
# Save

# Your site will be available at:
# https://YOUR_USERNAME.github.io/care-commons-docs/

Option 2: GitHub Actions Automation

Create .github/workflows/docs-deploy.yml in your docs repository:

name: Deploy Documentation

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Pages
        uses: actions/configure-pages@v4
      
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: '.'
      
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

Technical Details

Markdown Processing

  • Parser: marked library for GitHub-flavored markdown
  • Syntax highlighting: highlight.js for code blocks
  • Safe HTML: Marked sanitizes output by default

URL Slugs

File paths are converted to URL-safe slugs:

README.md                    ‚Üí readme.html
docs/DEPLOYMENT_GUIDE.md     ‚Üí docs-deployment-guide.html
packages/core/README.md      ‚Üí packages-core-readme.html

Navigation Limits

To maintain performance, navigation shows:

  • Up to 50 documents per category
  • "X more documents" indicator for additional files
  • All files are still generated and accessible

Search Implementation

Simple client-side search:

  • Searches navigation item text only
  • Case-insensitive matching
  • Shows up to 10 results
  • No full-text indexing (keeps it simple and fast)

For full-text search, consider integrating:

  • Pagefind
  • Algolia DocSearch
  • Lunr.js

Troubleshooting

"Cannot find module 'marked'"

Install dependencies:

npm install marked @types/marked --save-dev

Generated site has broken links

Ensure all internal markdown links use relative paths:

# ‚úÖ Good
[Link](./other-doc.md)
[Link](../parent/doc.md)

# ‚ùå Bad (absolute paths won't work)
[Link](/absolute/path/doc.md)

Search not working

Check browser console for errors. Common issues:

  • JavaScript disabled
  • Ad blocker interfering
  • Missing DOM elements

Site looks broken on mobile

Clear browser cache and check:

  • Viewport meta tag is present
  • CSS media queries are loading
  • Mobile menu JavaScript is executing

Too many documents, site is slow

Consider splitting into multiple sites or:

  1. Reduce navigation limits in generateNavigation()
  2. Implement pagination
  3. Add lazy loading for content
  4. Use a static site generator (VitePress, Docusaurus)

Advanced: Continuous Deployment

Automatic Updates from Main Repo

Set up a GitHub Action in your docs repository to pull updates:

name: Update Docs from Main Repo

on:
  schedule:
    - cron: '0 0 * * *'  # Daily at midnight
  workflow_dispatch:       # Manual trigger

jobs:
  update-docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          repository: YOUR_ORG/care-commons
          path: source
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'
      
      - name: Install dependencies
        run: |
          cd source
          npm install
      
      - name: Generate docs site
        run: |
          cd source
          npx tsx scripts/generate-docs-site.ts
      
      - name: Copy to docs repo
        run: |
          rm -rf !(source)
          cp -r source/docs-site/* .
          rm -rf source
      
      - name: Commit and push
        run: |
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add .
          git commit -m "Update documentation (automated)" || exit 0
          git push
